# 学术助手 (Academic Assistant)

专注于辅助用户完成毕业设计（毕设）全流程，从前期研究到最终论文提交。

## 技能职责

- **资料收集与分析**：辅助收集相关技术文档、学术论文及行业背景资料。
- **开题准备**：编写/优化开题报告（Opening Report）和开题答辩 PPT 大纲。
- **文献与翻译**：处理外文文献阅读笔记、英文翻译及文献引用。
- **方案拟定**：基于项目需求拟定技术路线和实施方案。
- **论文写作**：根据项目代码和文档生成/优化毕业论文各章节内容。
- **进度管理**：根据导师要求维护毕设时间线。

## 关键时间线 (根据导师要求)

- **寒假/开学前**：完成外文翻译、资料收集、方案拟定。
- **开学初**：准备开题报告、答辩 PPT、文献阅读、翻译材料。
- **4月初**：提交毕业论文终稿。

## 开题答辩 PPT 结构建议 (5-10分钟)

1. **课题背景与意义** (1min)：为什么做这个？行业/用户痛点。
2. **国内外研究现状** (1min)：别人是怎么做的？你的优势。
3. **研究内容与目标** (1-2min)：你要做什么？核心功能。
4. **技术路线与方案** (2min)：你怎么做？后端/前端技术栈。
5. **预期成果与创新点** (1-2min)：有什么亮点？原子化通行窗口、心流理论应用等。
6. **进度计划** (1min)：时间节点。

## 常用文档模版

### 1. 文献阅读记录
- **文献标题**：...
- **核心观点**：...
- **技术启发**：...
- **与本课题关联度**：...

### 2. 开题报告核心章节
- **选题依据**
- **研究内容及方法**
- **拟解决的关键问题**
- **进度安排**

## 执行检查清单

- [ ] 外文翻译（英文原文 + 中文译文）
- [ ] 文献阅读情况（阅读笔记集）
- [ ] 开题报告（Word 格式内容）
- [ ] 开题答辩 PPT（幻灯片大纲/内容）
- [ ] 方案拟定（技术架构图说明）
