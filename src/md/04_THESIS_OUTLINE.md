**适用专业**: 软件工程 / 计算机科学与技术
**核心关键词**: Spring Boot, 动态令牌, 抢占式熔断, 沉浸式助学, O2O

## 第一章 绪论 (Introduction)
### 1.1 研究背景与意义
*   **行业痛点**: 传统付费自习室人力成本高，无人值守模式下存在尾随、占座冲突等安全隐患。
*   **用户痛点**: 备考群体（考研/考公）普遍存在焦虑、孤独、注意力涣散等心理问题。
*   **研究意义**: 提出一种“技术管控+情感抚慰”的双向解决方案，具有商业应用价值和社会价值。

### 1.2 国内外研究现状
*   **门禁系统**: 现有的二维码门禁多为静态或简单刷新，缺乏高并发下的冲突处理机制。
*   **助学软件**: 专注类App（如Forest）与线下空间割裂，缺乏场景联动。

### 1.3 论文主要工作
*   构建基于 Spring Boot 的微服务后端。
*   设计软硬协同的动态二维码采集与鉴权方案。
*   实现基于倒计时熔断的防冲突算法。
*   开发沉浸式白噪音与专注计时模块。

## 第四章 系统概要设计 (System Design)
### 4.1 系统总体架构
*   采用 B/S 与 C/S 混合架构。
*   *(此处需插入系统架构拓扑图)*
### 4.2 核心算法与机制设计 **(论文核心亮点)**
*   **动态令牌协同机制**: Python Client 与 Java Server 的“生产者-消费者”模型。
*   **原子化通行窗口算法 (Atomic Passing Window)**:
    *   基于内存状态机的抢占式锁定逻辑。
    *   3分钟冷却机制对“防尾随”审计的逻辑支撑。
*   **智能轮询机制 (Smart Polling)**: 前端根据状态自适应调整请求频率的策略。
### 4.3 数据库设计
*   ER图设计。
*   数据表结构 (users, focus_records, user_pets)。

## 第七章 总结与展望 (Conclusion)
### 7.1 工作总结
*   完成了从门禁管控到助学服务的全链路闭环。
*   解决了无人值守场景下的核心痛点。
### 7.2 创新点
*   提出了“原子化通行窗口”概念，软硬件解耦。
*   融合了心流理论的情感化交互设计。
### 7.3 不足与展望
*   目前采用内存锁，未来可升级为 Redis 分布式锁。
*   未来将接入真实的红外/视觉传感器实现自动报警。
*   电子宠物与商业化功能的进一步迭代。
