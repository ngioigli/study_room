# 毕业设计开题报告内容稿：云端自习室智能管理与助学平台设计

## 1. 课题背景与意义

### 1.1 研究背景
随着考研、考公及各类职业资格考试竞争的加剧，付费自习室作为一种新型的共享空间在城市中迅速普及。然而，现有的自习室管理模式面临两个核心挑战：
1. **管理成本与秩序冲突**：无人值守模式下，如何低成本、高效率地管理门禁，并杜绝占座冲突、尾随入内等问题。
2. **用户心理关怀缺失**：备考群体普遍处于高压力、孤独的学习状态。现有的自习室仅提供物理空间，缺乏对用户学习效率和心理状态的正向引导。

### 1.2 研究意义
本课题旨在设计并实现一套“硬核管理+柔性助学”的 O2O（Online to Offline）平台。通过技术手段降低管理成本，通过游戏化设计提升用户粘性，实现自习室从“单纯的空间提供者”向“智慧助学伙伴”的转型。

---

## 2. 国内外研究现状

### 2.1 门禁管理现状
目前的二维码门禁系统多采用静态或低频刷新的动态码，容易被截屏传播。在无人值守场景下，当多名用户同时出现在门前时，缺乏明确的“抢占式”反馈机制。

### 2.2 助学类产品现状
市面上已有的助学 App（如 Forest、专注清单等）侧重于手机内的专注管理，与线下学习空间的联动较弱。用户在进入物理空间后，往往需要切换多个 App 来完成开门、计时、听白噪音等操作，体验碎片化。

---

## 3. 研究内容与目标

### 3.1 核心研究内容
1. **抢占式资源锁定机制**：设计“原子化通行窗口”算法，确保在高并发环境下，门禁资源能有序分配给单个用户。
2. **沉浸式助学环境构建**：利用 HTML5 Audio 接口实现多轨白噪音混音，并结合 CSS 呼吸动画设计低干扰的专注 UI。
3. **游戏化激励体系**：设计一套宠物养成系统，将抽象的学习时长转化为可视化的经验值与宠物形态进化。
4. **全链路状态同步**：实现 Python 客户端采集、Java 服务端中转、Web 前端展示的动态二维码分发链。

### 3.2 预期目标
开发一套包含后端管理、移动端入口、自动化采集端的完整系统，验证“一人一码、限时锁定”在防冲突场景下的有效性。

---

## 4. 技术路线与实施方案

### 4.1 技术栈选型
- **后端**：Spring Boot 2.6.13, MyBatis-Plus, MySQL 8.0。
- **前端**：HTML5, CSS3, JavaScript (ES6+), Vue 3 (CDN), ECharts。
- **架构**：B/S 架构，前后端分离设计，接口遵循 RESTful 风格。

### 4.2 核心算法：原子化通行窗口 (Atomic Passing Window)
- **锁定逻辑**：当用户请求开门时，后端数据库将二维码资源标记为 `LOCKED`，有效期 3 分钟。
- **排他性**：在锁定期间，其余用户看到的二维码处于模糊状态或显示“通道占用中”，从逻辑上杜绝多人同时尝试扫码带来的硬件冲突。

### 4.3 助学功能实现
- **白噪音**：支持 4-6 种环境音效（如雨声、篝火）的自由混合，提供独立音量调节。
- **宠物系统**：基于 Vue 3 响应式状态管理，实时根据专注时长结算 EXP，驱动宠物形态（蛋/幼年/成年）转换。

---

## 5. 课题创新点

1. **逻辑防冲突**：不同于传统的物理闸机排队，本系统在软件层实现了“先到先得”的抢占式窗口，降低了对硬件精度的依赖。
2. **心流化设计**：将专注、白噪音、宠物激励整合在同一个 Web 容器中，减少用户在学习时的设备切换频率。
3. **软硬解耦**：利用 Python 模拟网关，将线下的小程序/硬件输出转化为云端 API，具备极强的部署兼容性。
